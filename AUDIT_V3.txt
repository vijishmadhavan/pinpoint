CRITICAL (14) — Must fix, security/correctness/data-loss risk
=============================================================================

#1  [SEC] api/__init__.py:36 — Timing attack on API_SECRET
    Plain `!=` comparison allows character-by-character brute force.
    FIX: Use hmac.compare_digest(token, API_SECRET)

#2  [SEC] api/transform.py:841-929 — /run-python has zero sandboxing
    exec() with full __builtins__. Any code can rm -rf, exfiltrate data, etc.
    Combined with #1, this = full RCE if API_SECRET is brute-forced.
    FIX: Document as intentionally unrestricted OR add subprocess sandbox.

#3  [SEC] api/transform.py:806 — SSRF bypass via HTTP redirect
    _check_url_safe validates initial URL, but urllib follows 302→private IP.
    FIX: Use custom redirect handler that re-checks each hop.

#4  [SEC] api/helpers.py:154 — SSRF DNS rebinding via pass on gaierror
    DNS failure = pass. Attacker: public IP on first lookup, private on second.
    FIX: Reject on DNS failure (raise 400).

#5  [SEC] api/data.py:444-446 — pd.eval blocklist is bypassable
    df.to_csv('/tmp/evil') not blocked (no os./subprocess). Blocklist approach
    is fundamentally incomplete. FIX: Whitelist-based approach or df method blocklist.

#6  [BUG] face_search.py:296-313 — crop_face bbox mismatch
    Detection runs on resized image (1280px), crop applied to original (5000px).
    Wrong crop area guaranteed for images > MAX_FACE_DIM.
    FIX: Scale bbox coordinates back to original dimensions.

#7  [BUG] bot/index.js:98-99 — memoryEnabled is global, not per-chat
    User A's /memory off disables memory for ALL concurrent users.
    FIX: Make memoryEnabled a Map<chatJid, boolean>.

#8  [BUG] bot/index.js:1826-1853 — Reminder array splice during iteration
    splice(idx,1) shifts indices; concurrent push from set_reminder races.
    FIX: Collect indices to remove, splice in reverse order after loop.

#9  [BUG] image_search.py:107-116 — _get_conn() not thread-safe
    _db_lock exists but _get_conn() doesn't acquire it. Race on connection creation.
    FIX: Acquire _db_lock inside _get_conn().

#10 [BUG] photo_cull.py:58-61 — _get_conn() creates NEW connection every call
    Each call to _get_conn() → get_db() → new connection. Background threads with
    10 workers = 10+ leaked connections. FIX: Cache connection like image_search.

#11 [DATA] database.py:460-464 — chunk_document DELETE before INSERT not transactional
    If chunker.chunk() fails after DELETE commits, chunks are lost permanently.
    FIX: Wrap DELETE+INSERT in explicit transaction (BEGIN/COMMIT).

#12 [TEST] tests/test_audit_validation.py:12-16 — Tests hit PRODUCTION database
    client fixture doesn't patch DB_PATH. Writes to real memories table.
    FIX: Use conftest.py's client fixture that patches _get_conn.

#13 [CI] .github/workflows/test.yml:23-26 — Missing critical CI dependencies
    chonkie, insightface, onnxruntime, pytesseract, pillow-heif not installed.
    test_audit_validation.py imports face_search/image_search → ImportError in CI.

#14 [BUG] bot/index.js:679-709 — No timeout on API fetch calls
    If Python API hangs, bot hangs forever. activeRequests lock held = chat dead.
    FIX: Add AbortController with 120s timeout to all fetch calls.

HIGH (21) — Should fix, stability/correctness risk
=============================================================================

#15 [SEC] api/files.py:107-118 — Command injection incomplete sanitization
    Missing \r \n (newline injection). Linux find path: glob chars * ? [ ] not sanitized.
    FIX: Use subprocess with list args instead of string interpolation.

#16 [SEC] api/memory.py:186 — FTS injection via unescaped quotes
    Keywords with " break FTS5 syntax. FIX: Strip/escape " from keywords.

#17 [SEC] api/google.py:187 — Drive query injection via single quotes
    User query with ' breaks Drive API query syntax. FIX: Escape single quotes.

#18 [BUG] api/transform.py:893-904 — SIGALRM only works on main thread
    FastAPI runs endpoints in threadpool → ValueError: signal only works in main thread.
    /run-python timeout is completely broken in production.
    FIX: Use threading.Timer or subprocess with timeout.

#19 [BUG] face_search.py:387 — find_person reference image not preprocessed
    Reference at full resolution vs folder scans at 1280px → embedding mismatch.
    FIX: Apply _preprocess_for_face to reference image too.

#20 [LEAK] api/transform.py:367-396 — resize_image leaks pre-resize Image
    img.resize() creates new object, old img overwritten without close.
    FIX: Store original, close it after resize.

#21 [LEAK] api/transform.py:1101-1122 — _ocr_pdf_page leaks PIL Image
    img from Image.frombytes never closed. No try/finally.
    FIX: Add try/finally with img.close().

#22 [LEAK] extractors.py:99-100 — BytesIO not closed in _ocr_gemini
    Each page creates BytesIO buffer never closed. Accumulates in large batches.
    FIX: Close buffer after use.

#23 [LEAK] api/data.py:494-512 — pdfplumber not closed on page range parse error
    ValueError in int() → pdf.close() skipped. Same in pdf_to_excel (576-615).
    FIX: Use context manager (with pdfplumber.open(path) as pdf:).

#24 [BUG] api/files.py:649 — batch_move commits inside loop
    Partial failure: some files moved on disk + committed, rest rolled back.
    File system and DB become inconsistent.
    FIX: Collect all moves, then commit once at end.

#25 [BUG] api/files.py:816-818 — batch_rename doesn't update DB paths
    Unlike move_file, rename doesn't update documents/video_embeddings/photo_classifications.
    FIX: Add same path-update logic as move_file.

#26 [BUG] api/memory.py:429 — memory_delete doesn't clean FTS index
    Orphaned FTS entries return results for deleted memories.
    FIX: Call _memory_fts_sync or DELETE from memories_fts.

#27 [BUG] bot/index.js:1177 — compactHistory races with saveMessage
    Fire-and-forget during runGemini. DB reset while messages being saved.
    FIX: Await compactHistory before continuing, or use a mutex.

#28 [BUG] bot/index.js:1317 — response.candidates[0].content no null check
    Gemini safety block → empty candidates → TypeError crash.
    FIX: Guard with optional chaining: response.candidates?.[0]?.content

#29 [BUG] bot/src/tools.js:1733-1748 — preValidate misses second path
    compare_faces: only image_path_1 validated. merge_pdf/images_to_pdf: paths array
    not validated (checks args.path which is undefined).
    FIX: Add array path validation for tools that use paths[].

#30 [PERF] api/helpers.py:20-52 — Thread-local DB connections never closed
    Connections accumulate per thread, never released. File descriptor leak.
    FIX: Add connection cleanup on thread exit or use connection pool.

#31 [PERF] database.py:32 — init_db runs full schema migration every call
    search.py calls init_db() on every search → redundant schema checks.
    FIX: Cache init status, only migrate once.

#32 [BUG] photo_cull.py:1131-1159 — Moved files leave stale DB cache
    After group_photos moves files, photo_classifications/photo_scores still
    reference old paths. Same in cull_photos (447-464).
    FIX: Update paths in DB after move, or invalidate cache entries.

#33 [BUG] video_search.py:42-50 — _get_conn() not thread-safe
    Same as image_search.py #9. No lock on global connection creation.
    FIX: Add lock.

#34 [BUG] bot/index.js:1643-1657 — Temp file leak from image resize
    /tmp/pinpoint_send_*.jpg never cleaned up. Each oversized image = leaked file.
    FIX: Delete temp file after send completes (in finally block).

#35 [TEST] test_faces/photos/media — 100% mocked, zero integration testing
    All face/photo/media tests patch everything. Can't catch real bugs.
    Mocks have no spec=True → signature changes go undetected.
    FIX: Add spec=True to all mocks. Add at least one integration test per module.
